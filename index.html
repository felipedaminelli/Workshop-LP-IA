Fundamento

O que é uma Landing Page

Landing page é uma página com foco em uma ação principal. Ela remove distrações e guia a pessoa para uma conversão: preencher um formulário, pedir contato, baixar um material, solicitar demo ou se inscrever em uma campanha.

TráfegoAnúncio, e-mail, social, evento, orgânico ou remarketing.
MensagemPromessa clara, oferta relevante e prova de confiança.
ConversãoCTA + formulário + confirmação do próximo passo.
Fundamento

Por que usar uma LP

Foco

Uma página, uma campanha e uma ação principal. Isso reduz dispersão.

Medição

Fica mais fácil medir conversão, origem do lead e qualidade do tráfego.

Velocidade

Permite testar oferta, mensagem, CTA e formulário sem mexer no site inteiro.

Personalização

A LP pode falar diretamente com a dor de um público ou campanha.

Aprendizado

Cada campanha gera dados para melhorar a próxima versão.

Escala

O template vira base para novas campanhas e variações.

Briefing

O briefing

Antes de abrir o HubSpot ou pedir algo para a IA, o briefing precisa deixar claro o que a página deve comunicar, para quem e com qual ação principal.

Regra da aula: briefing bom não precisa ser longo; precisa ser claro o suficiente para orientar design, copy, formulário e prompts.

Nome da campanha: Novo pacote integrado Eberick + Builder + Visus

Estrutura obrigatória da LP: 4 seções

1. Hero:
Headline exemplo: Conecte Eberick, Builder e Visus em um fluxo integrado de projeto
Subheadline exemplo: Um pacote para aproximar estrutura, orçamento e visualização, reduzindo retrabalho e acelerando decisões.
CTA exemplo: Quero conhecer o pacote

2. Benefícios por produto:
Texto de abertura exemplo: Cada produto atua em uma etapa essencial do processo, criando uma experiência mais conectada do projeto à apresentação.

3. Como funciona o pacote:
Headline exemplo: Do cálculo à apresentação, tudo conversa melhor

Passo 3: Use o Visus para visualizar, comunicar e vender melhor a solução.

4. Conversão/Formulário:
Headline exemplo: Veja como o pacote pode funcionar no seu cenário

Atividade 2

Briefing do módulo com formulário

O briefing do módulo é igual ao da LP nativa. A diferença é que agora vamos encapsular a seção de conversão em um módulo customizado.

O módulo precisa ter

  • Seção completa de conversão.
  • Texto de apoio editável.
  • Lista curta de benefícios.
  • Imagem opcional.
  • Campo Form para selecionar formulário no HubSpot.
  • Formulário usado na aula: Teste-form-lp+IA.

Variáveis pensadas para HubSpot

  • anchor_id
  • eyebrow
  • title
  • description
  • benefits repetidor
  • form_field do tipo Form
  • privacy_note
  • spacing e estilos
Atividade sugerida: usar IA para gerar o módulo, colar no Design Manager e selecionar o formulário no editor da LP.

VS Code

VS Code + plugins

Fluxo sugerido

  • Criar pasta do exercício.
  • Separar module.html, module.css, module.js, fields.json e meta.json.
  • Usar Live Server para testar HTML/CSS estático.
  • Usar Gemini Code Assist ou Codex para revisar e refatorar.
  • Adaptar trechos HubL no Design Manager.

Cuidados

  • Não testar HubL puro como se fosse HTML comum.
  • Não usar biblioteca externa sem necessidade.
  • Escopar CSS por wrapper.
  • Conferir responsividade antes de publicar.
  • Validar o módulo dentro de uma LP real.
Atividade 3

Componente: seção de depoimentos

O componente será uma seção editável de depoimentos com cards que passam horizontalmente e pausam ao passar o mouse.

Comportamento

  • Cards em movimento contínuo.
  • Hover pausa a animação.
  • Foco por teclado também pausa.
  • Sem biblioteca externa.
  • Respeita prefers-reduced-motion.

Conteúdo do card

  • Texto do depoimento.
  • Foto da pessoa.
  • Nome da pessoa.
  • Cargo/empresa opcional.
  • Acento visual por produto: Eberick, Builder, Visus ou neutro.
Atividade sugerida: gerar o componente com IA, colar os arquivos no Design Manager e editar depoimentos pelo painel da página.
